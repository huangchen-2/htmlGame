在当前 C++ 生态中，Qt 几乎是桌面 GUI 开发的事实标准。无论是传统的工业软件，还是新兴的嵌入式 HMI，Qt 的身影无处不在。

## 为什么选 Qt

Qt 最大的优势是**跨平台**。一套代码，Windows、Linux、macOS 甚至 Android/iOS 都能跑。对于小团队或个人开发者来说，这省去了大量适配成本。

其次，Qt 不只是 GUI 框架——它是一整套 C++ 扩展库：

| 模块 | 用途 |
|------|------|
| Qt Core | 字符串、容器、多线程、文件IO |
| Qt Widgets | 传统桌面控件 |
| Qt Quick/QML | 现代触控 UI |
| Qt Network | HTTP/TCP 网络通信 |
| Qt SQL | 数据库操作 |
| Qt SerialPort | 串口通信（工控必备） |

## 三大方向

### 1. 桌面应用开发

最传统的方向，做 Windows/Linux 桌面软件。典型产品：

- 工业上位机（配合 PLC、传感器）
- 音视频处理工具
- 企业内部管理系统
- 数据可视化平台

这个方向稳，需求量大，但竞争也相对激烈。**推荐扎实掌握 Model/View 架构、多线程（QThread）、自定义控件绘制。**

### 2. 嵌入式 HMI

这是近年来增长最快的方向。车载中控、智能家电屏幕、医疗设备界面，大量使用 Qt for MCU 或 Qt Quick。

嵌入式 Qt 的特点是：
- 资源受限，需要关注内存和启动速度
- UI 以 QML 为主，Widgets 太重
- 和硬件交互频繁，需要了解 SPI、I2C、CAN 等通信协议

如果你对硬件感兴趣，这个方向非常值得深耕。

### 3. 游戏/仿真工具

虽然不是主流，但 Qt 在游戏开发工具链中很常见——关卡编辑器、资产浏览器、性能分析面板等。

对于想进游戏行业但不想直面渲染管线的 C++ 开发者来说，这是一个不错的切入点。

## 学习路线建议

1. **基础阶段**：C++11/14 扎实 → Qt Core → 信号槽机制 → QWidget 常用控件
2. **进阶阶段**：Model/View → 多线程 → 网络编程 → 自定义控件 → QML
3. **项目实战**：至少做 2-3 个完整项目，从需求到打包发布全流程
4. **深入方向**：根据目标行业选择嵌入式/桌面/游戏工具

## 找工作的几点建议

- **简历上要有能跑的项目**，不是 demo，是能实际使用的软件
- **开源贡献加分**，Qt 本身和 KDE 生态都是好的起点
- **工业/医疗/汽车**三个行业对 Qt 需求最大，薪资也相对高
- 不要只写 Qt——**数据库、网络协议、设计模式**同样重要

---

Qt 开发不是一个"火"的领域，但它是**稳定、持久、越老越值钱**的方向。不像前端框架年年换，Qt 30 年了还在用，未来 10 年也不会消失。
